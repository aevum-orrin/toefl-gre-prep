# TOEFL Learning — 2-Week Sprint (2026 New Format)

个人托福冲刺仓库。目标：两周内适配 **2026 年 1 月改版**的新托福，把口语 / 写作两块短板补上，稳住阅读听力。

> 底子：两年前托福 107/120（旧满分制）。本轮目标见 [plan/14-day-plan.md](plan/14-day-plan.md)。

## 为什么要重新学：2026 新托福 vs. 我两年前考的版本

我 2024 年考的已经是 **2023-07 改版**后的格式（缩短到 ~2h、写作换成 "Writing for an Academic Discussion"）。所以对我来说 **真正的新东西是 2026-01-21 的这波改革**：

| 维度 | 2023-07 版（我考过的） | **2026-01 版（本轮要练的）** |
|------|----------------------|------------------------------|
| 总时长 | ~116 分钟 | **~90 分钟** |
| 阅读 / 听力 | 固定题 | **多阶段自适应（adaptive）**——上一部分做得好，下一部分更难 |
| 口语 | 4 个任务 | **改成 2 类新任务：Listen and Repeat + Take an Interview（模拟面试）** |
| 计分 | 0–120 | **新增 1–6 band（对齐 CEFR），四科平均**；0–120 仍保留 |
| 出分 | — | **72 小时内** |

**结论**：口语题型是全新的（跟读 + 模拟面试），必须专门练；阅读听力要适应自适应难度爬升；写作沿用 Academic Discussion。

## 与 GRE 共享代码的架构（关键设计）

托福和 GRE 用**两个独立仓库**，但共享一个第三方内核仓库 **`prep-core`**（详见
[plan/14-day-plan.md](plan/14-day-plan.md) 的"可复用架构"一节）。凡是与考试无关的能力都下沉到 `prep-core`：

- **AI 反馈引擎** — 封装 Claude API，按传入的评分 rubric 给作文/口语打分与润色
- **SRS 背词** — 间隔重复算法（托福词表 / GRE 词表只是不同数据）
- **录音 + 转写** — 麦克风录音、ASR 转文字，供口语跟读/评分
- **进度追踪** — 练习记录、错题本

托福/GRE 各自仓库只放**考试特有的东西**：题型模板、rubric、词表、UI 文案。
